    Introdução

    Essa página tem como objetivo verificar os artefatos da Quarta Entrega do grupo 8.

    Metodologia

    Nessa página sobre a entrega 2 é possível se observar a verificação de todos os artefatos dessa mesma entrega sendo eles Backlog, Histórias de Usuário, NFR Framework . A verificação foi realizada utilizando a Tabela 1 como template.

    Tabela 1: Template para verificação

    Critérios Avaliação Fonte
    Pergunta para avaliação Sim/Não/Incompleto Página e livro de referência

    Comentários

    Os comentários relacionados a cada artefato serão detalhados aqui.

    Backlog

    • Aqui é possível encontrar a origem do artefato verificado Backlog.

    Tabela 2: Verificação do Backlog

    Critérios Sim/Não/Incompleto Fonte
    1 - Existe apenas um backlog mantido para o projeto inteiro ? Sim p.387
    2 – O backlog inclui Épicos, histórias de usuário e priorização ? Incompleto p.387
    3 - Os itens do backlog são rastreáveis até os requisitos de origem para facilitar a priorização ? Sim p.387
    4 - Existe uma distinção clara entre histórias de usuário e épicos, sendo os épicos definidos como histórias grandes demais para serem implementadas em uma única iteração ? Sim p.388
    5 - As prioridades dos itens do backlog estão sendo gerenciadas de forma eficaz ? Não p.387
    6 - As prioridades atribuídas aos itens do backlog são flexíveis e podem mudar nas próximas iterações caso necessário ? Sim p.388
    7 - As histórias de usuário servem como ponto de partida para conversas que detalham ainda mais os requisitos ? Sim p.388

    Autor: Johnny Lopes & Paulo Borba

    Vídeo 1 - Verificação do backlog

    Autor: Johnny Lopes.

    Comentários

    • Seria interessante adicionar hiperlinks na tabela de backlog.
    • No artefato, não é informado como a priorização foi feita, nem qual técnica foi utilizada.

    Histórias de Usuário

    • Aqui é possível encontrar a origem do artefato verificado Histórias de Usuário.

    Tabela 3: Verificação das Histórias de Usuário

    Critérios Sim/Não/Incompleto Fonte
    1 - As histórias de usuário foram escritas pelo cliente ou foram feitas utilizando os requisitos já elicitados? Sim p.73
    2 – As histórias de usuário possuem um valor (uma prioridade) baseado no valor de negócio dada pelo cliente? Sim p.73
    3 - No artefato, cada história de usuário possui um identificador único? Sim Fonte
    4 - As descrições das histórias de usuário são claras e descrevem bem as funcionalidades que o sistema deve possuir para solucionar o problema do usuário? Sim p.73
    5 - As histórias de usuário descrevem o resultado, as características e as funcionalidades solicitados para o software a ser construído? Sim p.73
    6 - As histórias de usuário possuem uma área dedicada para a rastreabilidade dos requisitos que foram utilizados em cada história? Não Fonte

    Autor: Gabriel Souza & José Filipi

    Cenários

    • Aqui é possível encontrar a origem do artefato verificado Cenários.

    Tabela 4: Verificação dos Cenários

    Critérios Sim/Não/Incompleto Fonte
    1 - Cada cenário descreve uma única instância de uso do sistema? Sim p.149
    2 – Para cada cenário, há uma referência do caso de uso que foi especializado no cenário? Não p.149
    3 - Os cenários contém declarações gerais dos objetivos do usuário ou tarefas de negócios que os usuários presisam realizar? Sim p.136
    4 - Os episódios de cada cenário seguem uma ordem lógica de acontecimentos Sim p.152
    5 - Os cenários especificam um caso de uso? Incompleto p.163

    Comentários

    • Nos cenários criados pelo grupo, não é possível identificar qual caso de uso foi especificado por cada cenário, seria interessante a utilização de hyperlinks.

    Autor: Gabriel Souza & José Filipi

    NFR FrameWork

    • Aqui é possível encontrar a origem do artefato verificado NFR Framework .

    Tabela 5: Verificação do NFR Framework

    Critérios Sim/Não/Incompleto Fonte
    1 - Os requisitos não funcionais foram identificados e documentados ? Sim p.19
    2 – Foi utilizada a abordagem goal-oriented para a especificação dos requisitos não funcionais ? Sim p.19
    3 - Os softgoals foram definidos e classificados de acordo com o NFR Framework ? Sim p.20
    4 - Foi utilizado o Softgoal Interdependency Graph (SIG) como modelo para representar as interdependências entre os softgoals ? Sim p.24
    5 - Os softgoals foram priorizados de acordo com a importância para o sistema ? Sim p.25
    6 - Foram identificados e documentados os possíveis conflitos entre os softgoals ? Sim p.180
    7 - Foi elaborado um plano para gerenciar e resolver os conflitos identificados ? Sim p.180
    8 - Foi elaborado um plano de ação para implementar os softgoals escolhidos ? Sim p.44
    9 - Foram elaborados cartões de especificação para os requisitos não funcionais modelados ? Sim p.45

    Autor: José Filipi & Vitor Leonardo
